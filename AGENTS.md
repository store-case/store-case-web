# 🚀 프론트엔드 코드 컨벤션 및 클린 코드 가이드라인 (agents.md)

본 문서는 프로젝트의 모든 **프론트엔드 코드 (React 컴포넌트, JavaScript/JSX 파일)** 에 적용되는 주요 컨벤션 및 클린 코드 규칙을 정의합니다.

---

## 1️⃣ 일반 JavaScript 규칙

| 규칙 | 내용 | 예시 (나쁜 예 → 좋은 예) |
|------|------|---------------------------|
| **변수 선언** | 변경되지 않는 변수에는 **const**, 변경될 가능성이 있는 변수에는 **let**을 사용합니다. **var**는 사용하지 않습니다. | `var data = [];` → `const data = [];` |
| **화살표 함수** | 특별한 이유가 없는 한 **화살표 함수(=>)** 문법을 사용하여 간결성을 높입니다. | `function sum(a, b) { return a + b; }` → `const sum = (a, b) => a + b;` |
| **비구조화(Destructuring)** | 객체나 배열의 속성에 접근할 때 비구조화를 적극적으로 사용합니다. | `props.user.name` → `const { name } = props.user;` |
| **삼항 연산자** | 간단하고 명확한 조건부 로직에는 삼항 연산자를 활용하되, 복잡한 로직에는 `if/else` 구문을 사용합니다. | `if/else 블록` → `{ condition ? <A /> : <B /> }` |

## 2️⃣ React 컴포넌트 규칙

### 2.1 컴포넌트 정의 및 구조

| 규칙 | 내용 |
|------|------|
| **함수형 컴포넌트 사용** | 클래스 컴포넌트 대신 함수형 컴포넌트와 Hooks를 사용합니다. |
| **단일 책임 원칙 (SRP)** | 각 컴포넌트는 하나의 주요 책임만 갖도록 설계합니다. 로직이 커지면 커스텀 Hooks나 하위 컴포넌트로 분리합니다. |
| **Props 명확성** | 컴포넌트 함수 인자에서 props를 즉시 비구조화하여 어떤 속성이 사용되는지 명확히 합니다. |
| **타입 정의** | 컴포넌트 Props의 타입을 명시하기 위해 TypeScript 또는 **prop-types**를 필수로 사용합니다. |

### 2.2 Hooks 사용 원칙

| 규칙 | 내용 |
|------|------|
| **Hooks 순서** | `useState`, `useEffect` 등의 모든 Hooks는 컴포넌트 최상위에서 호출되어야 합니다. (React 규칙 준수) |
| **의존성 배열 관리** | `useEffect`, `useCallback`, `useMemo`의 의존성 배열(`[]`)은 항상 정확하게 채웁니다. ESLint의 `exhaustive-deps` 규칙을 준수합니다. |
| **커스텀 Hooks** | 반복되는 상태 관리나 로직은 `use`로 시작하는 커스텀 Hooks로 분리하여 재사용성을 높입니다. |

## 3️⃣ 명명 규칙 (Naming Conventions)

일관된 명명 규칙은 코드를 빠르게 이해하는 데 핵심적인 역할을 합니다.

| 대상 | 규칙 | 예시 (좋은 예) |
|------|------|----------------|
| **컴포넌트** | 파스칼 케이스 (PascalCase). 파일명과 일치시켜야 합니다. | `UserProfile.jsx`, `ModalContainer.jsx` |
| **Hooks** | `use` 접두사를 사용하는 카멜 케이스 (camelCase). | `useFetchData`, `useLocalStorage` |
| **일반 변수/함수** | 카멜 케이스 (camelCase). | `itemCount`, `handleInputChange` |
| **Boolean 변수** | `is`, `has`, `can` 등의 접두사를 사용하여 Boolean 값임을 명확히 합니다. | `isLoading`, `isModalOpen`, `hasError` |

## 4️⃣ 코드 스타일 및 포맷팅

| 규칙 | 내용 |
|------|------|
| **들여쓰기** | 스페이스 2칸 또는 4칸 중 하나로 통일합니다. (프로젝트 설정 확인) |
| **세미콜론** | 세미콜론 사용 여부는 프로젝트 전체에서 일관되게 유지합니다. *(권장: 사용)* |
| **문자열** | 문자열에는 **작은따옴표 (')**를 사용하고, 변수를 포함하는 경우 **백틱 (`)**을 사용한 템플릿 리터럴을 사용합니다. |
| **JSX 괄호** | 여러 줄에 걸친 JSX는 항상 **괄호 (())**로 감싸서 코드의 명확성을 높입니다. |
| **자동 포맷터** | Prettier와 같은 자동 포맷터를 사용하며, **커밋 훅 (Commit Hook)**을 통해 포맷팅을 강제하여 일관성을 유지합니다. |
